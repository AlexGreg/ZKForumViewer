Howto refresh a selectbox from Java within an ViewModel bound with BindComposer

stundzig
24 Jul 2012 08:05:28 GMT
24 Jul 2012 08:05:28 GMT

Hi all,

I'm on my first project with zk. And it works very good. Thank you comitters.

I've one question. I'm using the BindComposer and the ViewModels as in the tutorial. I've one page index.zul which includes a tabbox and two other pages. One of the pages, the category page contains a listbox and a edit area. The other page contains events, also with list and detail. The events-detail form has a selectbox, which contains a model with all available categories. All models and buttons are bound with @bind or @load directives.

And now my question. If I add a new category in the first page, how can i trigger the BindComposer of the events page to reload the model for the categories box?

I've tried different approaches, e.g. calling invalidate on the selectbox or firing an event from java

---
EventQueue<org.zkoss.zk.ui.event.Event> lookup = EventQueues
.lookup("$ZKBIND_DEFQUES");
Selectbox box = (Selectbox) Executions.getCurrent().getDesktop()
.getAttribute("events.selectLocation");

box.invalidate();
box.setModel(box.getModel());
lookup.publish(new PropertyChangeEvent(box.getRoot(), box, "locations"));
---

but nothing works. The selectbox isn't refreshed. If i add a button on the events page and add the Annotation @NotifyChange("locations") on the java method for the button, the selectbox is refreshed if i click it in the ui. But thats not very helpful.

Can anybody give me a hint please?

Thanks.

regards
