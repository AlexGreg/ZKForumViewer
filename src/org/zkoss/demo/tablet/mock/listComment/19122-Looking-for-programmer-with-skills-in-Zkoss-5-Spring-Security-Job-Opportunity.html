Job Opportunity!

We're looking for a experienced programmer knowledgable in ZKoss (5.8), Spring framework (3) + Spring Security to help us with a custom way of signing in users programmatically with support for "remember me".

Short description:
We need a solution for signing in user programmatically (handled in application as a ZK-event where user clicks a login-button) without ZK-Spring-Security intercepting the process launching modal popups with the login page. This sign in process should also be managed without the need for a page redirect (form post) which would be the default way of signing in a user through Spring-Security. The user shold stay on the same page. We also need to have full support for "remember me" for logged in users as well. Sign out should be managed by application as well and that without any ZK-Spring-Security pop-ups.

http://tinyurl.com/cds8veg

A resource/example project we've set up to use for solving the assignment (import into Eclipse as "Java Project", add Facet "Dynamic Web Project", then runnable in Tomcat 7 or the like):
http://tinyurl.com/cb9e74y

Without trying to put someone out of a job, have a look at SHIRO from Apache - it's a lightweight security system - no need for Spring. It works with ZK and Java no problem. Even works with ZK and Python if you want, but I guess you are talking Java?
